feat(core): recovery + TTL + переименование операций; упрощение хранилища

BREAKING CHANGES:
- Переименованы операции: cut -> cut_video, to_shorts -> make_short (extract_audio без изменений)
- Упрощено хранение: удалены подпапки input/output/temp; все файлы теперь лежат прямо в /app/tasks/{task_id}/
- Изменились пути скачивания: теперь /download/{task_id}/{filename} (без /output/)
- Семантические префиксы выходных файлов: short_*.mp4 (make_short), video_*.mp4 (cut_video), audio_*.mp3/.aac (extract_audio)

Recovery (восстановление зависших задач):
- Автовосстановление задач в статусе processing/queued при старте контейнера
- Опциональный периодический сканер (по переменной окружения RECOVERY_INTERVAL_MINUTES)
- TTL задач (TASK_TTL_HOURS, по умолчанию 2 часа): просроченные помечаются failed и удаляются
- Сохраняем валидные input_* файлы; temp_* очищаются перед повторной попыткой
- Лимит попыток и задержка между ними настраиваются (MAX_TASK_RETRIES, RETRY_DELAY_SECONDS)

Метаданные и TTL:
- В sync и async режимах metadata.json теперь содержит: created_at, expires_at, retry_count
- Это позволяет единообразно чистить задачи после истечения TTL и корректно восстанавливать незавершённые

Валидация перед FFmpeg:
- Добавлен метод validate_input_file(): проверка наличия и минимального размера (>1KB)
- Вызов перед каждой операцией (cut_video, make_short, extract_audio) — предотвращает «пустые» входы и неочевидные ошибки FFmpeg

Окружение (значения по умолчанию):
- RECOVERY_ENABLED=true — включает механизм восстановления
- RECOVERY_INTERVAL_MINUTES=0 — только при старте ( >0 — периодическое сканирование)
- MAX_TASK_RETRIES=3 — максимум повторов восстановления
- RETRY_DELAY_SECONDS=60 — задержка между повторами
- TASK_TTL_HOURS=2 — время жизни задач (используется для expires_at и автоочистки)

Структура задач (новая):
- /app/tasks/{task_id}/
  - input_*.mp4 — входные файлы
  - temp_*.mp4 — временные промежуточные файлы
  - short_*.mp4 | video_*.mp4 | audio_*.mp3 — финальные файлы
  - metadata.json — метаданные результата (с TTL полями)

URL и скачивание:
- download_path теперь формата: /download/{task_id}/{filename}
- download_url соответствует absolute URL согласно текущей базовой конфигурации

Логи старта:
- Выводят параметры Recovery (TTL, retries, delay, interval)

Тесты (локально):
- make_short (sync): создаёт short_*.mp4, metadata с expires_at ✓
- extract_audio (sync): создаёт audio_*.mp3, metadata с TTL полями ✓
- make_short (async): формирует download_path без /output, metadata обновляется ✓
- Recovery: стартовый скан запускается, ошибок нет ✓
- Валидация: операции не стартуют на отсутствующих/пустых файлах ✓
